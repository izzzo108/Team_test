# Онбординг: быстрый старт (5 минут)

Этот файл — для нового участника команды **и** для его Claude Code/Claude в VS
Code, который будет ему помогать. Если ты Claude и читаешь это в начале работы —
дальше по тексту есть отдельная секция специально для тебя.

## Что это за проект

Сайт с коллекцией маленьких пиксель-артовых игр (`games/<slug>/`). Настоящая цель
проекта — не игры, а тренировка команды в совместной работе через Git/GitHub:
ветки, Pull Request'ы, разрешение конфликтов. Подробно про идею — в [README.md](README.md).

## Что сделать перед первой задачей

1. **Установить git** (если ещё нет) и, по желанию, VS Code с Claude Code —
   расширение подхватит агентов/скиллы из `.claude/` автоматически, инструкции
   ставить не нужно.
2. **Склонировать репозиторий:**
   ```bash
   git clone https://github.com/izzzo108/Team_test.git
   cd Team_test
   ```
3. **Открыть `index.html`** в браузере — двойным кликом, без сервера. Убедиться,
   что список игр открывается и текущая игра работает.
4. **Прочитать** (по 2 минуты каждый):
   - [CLAUDE.md](CLAUDE.md) — стек и конвенции проекта.
   - [GAMES.md](GAMES.md) — как добавить свою игру, не задев чужие.
   - [README.md](README.md), раздел "Как работать с GitHub в команде" — ветки,
     коммиты, Pull Request.

## Твоя первая задача: одна игра — одна ветка

```bash
git checkout main
git pull origin main
git checkout -b feature/<название-твоей-игры>
```

Дальше — создаёшь `games/<slug>/` по инструкции из GAMES.md, коммитишь, пушишь
ветку и открываешь Pull Request на GitHub. Никто другой в это время не трогает
твою папку — конфликтов почти не будет.

## Секция для Claude Code (если ты — ассистент нового участника)

Коротко, что можно предложить человеку и что уже готово в проекте:

- **Агенты** (`.claude/agents/`): `architect` — продумать структуру перед большой
  фичей; `planner` — разбить задачу на шаги; `code-reviewer` — проверить код перед
  коммитом/PR; `code-simplifier` — упростить разросшийся код. Вызываются по
  запросу своими словами ("спланируй фичу X", "проверь код").
- **Скиллы** (`.claude/skills/`): `git-workflow` (ветки/коммиты/PR/конфликты),
  `coding-standards` (базовые правила кода), `frontend-design-direction`
  (визуальные решения). Подключаются автоматически по контексту разговора.
- **Архитектура сайта** уже спроектирована агентом `architect` (см. историю
  коммитов) — не нужно предлагать переход на фреймворк/сборщик/движок, проект
  сознательно остаётся на чистом HTML/CSS/JS без npm.
- Если пользователь хочет **добавить новую игру** — направь его сразу к
  [GAMES.md](GAMES.md), там пошаговая инструкция и шаблон `index.html`.
- Если пользователь не понимает, **что делать с git** — см. `git-workflow` скилл
  и раздел README "Как работать с GitHub в команде", не изобретай процесс заново.
- Не создавай общий код в `shared/`, пока он не дублируется в 2+ играх — см.
  правило в CLAUDE.md.
