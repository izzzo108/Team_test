# Проект: Pixel Team Games

Учебный сайт с коллекцией маленьких пиксель-артовых игр. Цель проекта — не сами игры,
а тренировка совместной работы через Git/GitHub в команде. Подробности процесса —
в README.md, как онбордиться новому участнику — в ONBOARDING.md, как добавить игру —
в GAMES.md.

## Стек

- Чистый HTML/CSS/JS, без сборки, без npm, без фреймворков. Только обычные
  `<script>`-теги (без `type="module"`) — проект открывается двойным кликом по
  `index.html`, без локального сервера.
- Каждая игра — отдельная папка `games/<slug>/` со своими `index.html`, `game.js`,
  `style.css`. Общий мини-движок — `shared/` (`loop.js`, `input.js`, `canvas.js`,
  `game.css`). Тема сайта (шапка/меню/цвета лендинга) — `theme.css`, отдельно от игр.

## Структура

```
index.html    # хаб — список игр (рендерится из games.js)
games.js      # реестр игр для главной страницы
theme.css     # тема сайта (лендинг), не для игр
GAMES.md      # как добавить новую игру
shared/       # общий мини-движок для игр (необязательный к использованию)
games/<slug>/ # одна игра — одна папка, полностью изолирована от других игр
templates/    # альтернативные варианты дизайна лендинга (финальный ещё не выбран)
```

## Конвенции

- Не создавайте общий код "на будущее" — в `shared/` попадает только то, что реально
  дублируется в 2+ играх.
- Игра не должна знать про `theme.css` и наоборот — это разные заботы (см. GAMES.md,
  раздел "Тема сайта").
- Комментарии — только там, где неочевидно "почему", а не "что".
- Коммиты и ветки — по правилам из `.claude/skills/git-workflow/SKILL.md` (GitHub Flow,
  Conventional Commits).
- Регистрируя новую игру, трогайте только свою строку в массиве `GAMES` (`games.js`) —
  это единственная общая точка записи между играми разных участников.
